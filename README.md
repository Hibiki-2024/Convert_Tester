# Convert_Tester

## 概要
`Convert_Tester` は、テキストを内部の変換表と暗号鍵を用いて別の文字列へ変換するサンプルツールです。主にブラウザ上で動作するシンプルなデモで、Unicode エスケープの処理や変換テーブルによるマッピングの挙動確認ができます。

## 特徴
- 変換ロジックは `ConvTes.js` に実装されています。
- 変換表は `ConversionData.js` の `CONVERSION_TABLE` を参照します（埋め込み形式）。
- ブラウザで簡単に動作を確認できます（ローカルファイルまたは簡易HTTPサーバで実行）。

## ファイル構成

- `index.html` - デモ用のHTML。ブラウザで開いて動作を確認します。
- `ConvTes.js` - 変換処理（Unicodeエスケープ→数値化→変換表マッピング等）。
- `ConversionData.js` - 変換テーブル（`CONVERSION_TABLE`）を定義。
- `ConversionData.js` のほか、テスト用のHTMLやCSSが多数含まれます（例: `TestHTMLEncode.html`, `TestHTMLDecode.html`, `TestHTML.css`）。

## 使い方

1. クローン（既にローカルにある場合はこの手順は不要です）。

```zsh
git clone <リポジトリURL>
cd Convert_Tester
```

2. ブラウザで `index.html` を直接開くか、簡易HTTPサーバを使って起動します（推奨）。

```zsh
# ポート8000で簡易サーバを起動（Python 3）
python3 -m http.server 8000
# ブラウザで http://localhost:8000/ を開き、index.html を表示
```

3. ブラウザのデベロッパーツール（コンソール）で変換結果やログを確認します。

## 開発メモ

- 変換ロジックの主な関数は `ConvTes.js` に集約されています。テスト用HTMLを編集して入力や鍵を変えることで挙動を確認できます。
- 逆変換（変換結果から元の文字列への復元）を試す実装もプロジェクトに含まれている場合があります。

## 注意事項

- ローカルで直接ファイルを開くと一部のブラウザでスクリプトやリソースの制限が働くことがあります。問題がある場合は簡易HTTPサーバでの起動を推奨します。
